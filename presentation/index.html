What does CDI (Contexts and Dependency Injection) do for us?
Context
  • Manage a bean's lifecycle
  • Bind a bean to a context
Dependency
  • Discover and resolve
  • Bean Metadata
  • Loose coupling
Injection
  • Injection
  • Strong typing
Life without CDI
  • constructor injection
  • setter injection
  • string based injection (Seam 2 / Spring)
  • JNDI lookups
  • Session / Request lookup and storage
  • Tight coupling to 3rd party libraries
Life with CDI
  • Minimal configuration
  • Compiler checked mistakes
  • Fail fast at deploy time
  • Standardized Injection
  • Multiple scopes
  • Very loose coupling to 3rd party libraries
